## Tooling
- Spaces over tabs. Doesn't matter that much, but be consistent.
- VS Code with vim keybindings. Not a hill to die on, just the correct answer.
- ORMs are fine until they're not. Know when to write the SQL yourself.

## Software
- Most teams don't have the scale that justifies microservices.
- 100% test coverage is usually theater. Test the things that break.
- TypeScript is good. Overengineered TypeScript is its own special problem.

## Games
- Turn-based JRPGs hit differently than action RPGs for storytelling.
- A game that respects your time is a good game regardless of genre.
